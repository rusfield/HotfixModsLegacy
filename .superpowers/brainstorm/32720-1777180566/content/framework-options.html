Framework Direction

Windows desktop first, with the architecture kept portable for Mac/Linux later.

A

MAUI + Blazor Hybrid

Best fit for JS packages, CSS themes, rich searchable dropdowns, Semantic Kernel chat UI, and reusing web UI patterns. Windows first is WebView2; later Mac uses Mac Catalyst.

B

Avalonia Desktop

Best fit for true native cross-platform desktop. Strong theming and Linux/Mac story, but searchable data-heavy editors and JS integration take more custom work.

C

Portable Core, Blazor First

Build the domain, MySQL, settings, events, and AI contracts UI-agnostic. Ship MAUI Blazor first, but keep Avalonia possible later without rewriting business logic.

Recommendation

Choose C in architecture and A for the first shell: a clean portable core with a MAUI Blazor Hybrid Windows app as the first UI.
